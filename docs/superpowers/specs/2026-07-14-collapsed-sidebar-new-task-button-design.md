# 折叠侧边栏新建任务按钮设计

## 目标

侧边栏展开时保留当前标题与新建任务按钮；侧边栏折叠后，在左上角标题位置显示“+”按钮。用户点击该按钮后进入 `/?action=new`，侧边栏继续保持折叠。

## 交互设计

- 展开状态：应用标题用于折叠侧边栏，右侧“+”用于新建优化任务。
- 折叠状态：仅显示居中的“+”，点击后新建优化任务，不展开侧边栏。
- 折叠态按钮沿用 `editor.newTask` 作为 `title`、`aria-label` 和提示文本。

## 实现方案

在 `AppSidebar` 中读取 `useSidebar().state`，根据展开状态显式渲染标题按钮或折叠态新建任务按钮。两种状态下的新建入口复用同一目标地址和点击处理函数，避免图标含义与实际行为不一致。

本次只调整侧边栏头部，不修改任务创建流程、路由参数或其他侧边栏内容。

## 验证

- 展开侧边栏时，标题和右侧“+”正常显示。
- 点击标题后侧边栏折叠，左上角显示“+”。
- 点击折叠态“+”后进入新任务页面，侧边栏保持折叠。
- TypeScript 与 ESLint 检查通过。
