# 登录页游客入口设计

## 目标

在登录页提供清晰的“随便看看”入口，让未登录用户无需填写表单即可进入文本优化工作台。

## 交互

- 入口仅在验证码登录和密码登录状态显示。
- 入口使用位于登录主按钮下方的描边按钮，中文文案为“随便看看”，英文文案为“Explore as guest”。
- 点击后直接跳转到当前语言下的首页 `/`。
- 跳转时不创建账号、不请求认证接口，也不清除已有游客 ID 或游客使用次数。
- 注册和重置密码状态不显示该入口，避免干扰当前操作。

## 数据流

首页已经允许匿名访问，文本优化功能会在访客首次使用时按需创建游客 ID。因此登录页只负责导航，不新增游客状态、接口或持久化逻辑。

## 国际化

在 `pages.login` 命名空间下新增游客入口文案，并保持中英文翻译文件结构一致。

## 测试与验收

- 验证游客入口仅在两种登录状态出现。
- 验证点击入口后跳转到首页。
- 验证该操作不会清除已有游客使用记录。
- 运行相关前端测试、类型检查和代码检查。
